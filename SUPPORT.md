## Support

If you need help using or extending **SLM Benchmark**, this document explains how to get support.

### Self-Service Resources

- **README.md** — high-level overview, quick start, and methodology.
- **DEVELOPER_GUIDE.md** — internals and extension points.
- **TROUBLESHOOTING.md** — common issues and fixes.

### Getting Help

- **Bug reports** — use GitHub Issues:
  - Include OS, Python version, exact command, full traceback, and any YAML used.
- **Feature requests & questions** — open a Discussion:
  - Share your use case and constraints (e.g. “CPU‑only GitHub Actions”, “offline environment”).
- **Model submission questions** — attach your YAML and link to the Hugging Face model page.
